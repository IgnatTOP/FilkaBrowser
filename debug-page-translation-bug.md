# Debug Session: page-translation-bug

Status: OPEN
Date: 2026-06-15

## Symptoms

- Непонятно, выполняется ли перевод вообще.
- Перевод продолжает работать, когда он уже не нужен.
- Результат перевода появляется только после перезагрузки страницы.

## Scope

- Investigate translator lifecycle, trigger conditions, and UI update path.
- Collect runtime evidence before changing translator logic.

## Hypotheses

1. Перевод запускается, но результат не попадает в активную модель/DOM до полного пересоздания страницы.
2. Нет корректной отмены или проверки актуальности задачи, поэтому старые переводы продолжают применяться после смены состояния.
3. UI-панель показывает устаревший статус и не отражает реальный ход перевода.
4. Сигналы/события завершения перевода не доходят до QML или не вызывают обновление отображения.
5. Переводчик повторно использует кэш/снимок страницы, а не текущий контент, из-за чего эффект виден только после reload.

## Plan

1. Найти все точки запуска, отмены и применения перевода.
2. Добавить минимальную инструментализацию.
3. Воспроизвести проблему и собрать логи.
4. Подтвердить гипотезу и внести минимальный фикс.
5. Проверить поведение после фикса и затем убрать отладку после подтверждения.

## Findings

- Кэш текста страницы жил отдельно от URL вкладки, поэтому панель могла считать старый текст "актуальным".
- При `loading/url` изменениях переводчик не сбрасывал состояние страницы целиком.
- Если пользователь нажимал перевод в момент загрузки, не было надежной цепочки `дождаться завершения -> заново извлечь текст -> запустить перевод`.
- Закрытие панели отменяло запрос, но не очищало состояние, из-за чего при повторном открытии мог показываться старый результат.
- `SidePanel` не перехватывал клики внутри drawer, поэтому тап по кнопке "Перевести" пробрасывался в scrim и просто закрывал панель.

## Fix Applied

- Добавлен `cachedUrl` в `PageTranslator`, чтобы валидировать кэш относительно текущей вкладки.
- `BrowserView` теперь сбрасывает переводчик при смене вкладки, URL и старте загрузки; после завершения загрузки повторно извлекает текст, если панель открыта.
- `TranslatorPanel` теперь работает только со свежим кэшем текущей страницы, умеет заново запрашивать извлечение и очищает состояние при закрытии.
- `SidePanel` теперь гасит клики внутри панели и не закрывается при нажатии на внутренние кнопки.
- Сохранена инструментализация для сбора runtime-логов, если после проверки пользователя понадобится второй проход.

## Verification

- `cmake -S . -B build && cmake --build build -j4` проходит успешно.
- Сессия остается OPEN до подтверждения пользователя; debug-инструментализацию пока не удалять.
